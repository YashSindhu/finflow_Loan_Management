SECTION 1: DOCKER COMMANDS
================================================================================

--- DAILY START ---
cd c:\Users\yashs\Desktop\finflow
docker-compose up -d

--- DAILY STOP ---
cd c:\Users\yashs\Desktop\finflow
docker-compose down

--- CHECK ALL CONTAINERS RUNNING ---
docker ps

--- VIEW LOGS OF A SERVICE ---
docker logs auth-service -f
docker logs application-service -f
docker logs admin-service -f
docker logs document-service -f
docker logs api-gateway -f
docker logs config-server -f
docker logs eureka-server -f

--- RESTART A SINGLE SERVICE ---
docker restart auth-service
docker restart application-service
docker restart admin-service
docker restart document-service
docker restart api-gateway

--- AFTER CODE CHANGES ---
mvn clean package -DskipTests -f auth-service/pom.xml
docker cp auth-service/target/auth-service-0.0.1-SNAPSHOT.jar auth-service:/app/app.jar
docker restart auth-service

mvn clean package -DskipTests -f application-service/pom.xml
docker cp application-service/target/application-service-0.0.1-SNAPSHOT.jar application-service:/app/app.jar
docker restart application-service

mvn clean package -DskipTests -f admin-service/pom.xml
docker cp admin-service/target/admin-service-0.0.1-SNAPSHOT.jar admin-service:/app/app.jar
docker restart admin-service

mvn clean package -DskipTests -f document-service/pom.xml
docker cp document-service/target/document-service-0.0.1-SNAPSHOT.jar document-service:/app/app.jar
docker restart document-service

mvn clean package -DskipTests -f api-gateway/pom.xml
docker cp api-gateway/target/api-gateway-0.0.1-SNAPSHOT.jar api-gateway:/app/app.jar
docker restart api-gateway

--- WIPE ALL DATA AND START FRESH ---
docker-compose down -v
docker-compose up -d

--- PUSH CODE TO GITHUB (triggers CI/CD) ---
git add .
git commit -m "your message here"
git push

SECTION 5: VALIDATION RULES
================================================================================

Password Rules:
  - Minimum 8 characters
  - At least 1 uppercase letter
  - At least 1 lowercase letter
  - At least 1 digit
  - At least 1 special character (@$!%*?&)
  - Example: Password@123

Phone Rules:
  - Must be a valid 10-digit Indian mobile number
  - Must start with 6, 7, 8, or 9
  - Example: 9876543210

Employment Types (exact values):
  SALARIED | SELF_EMPLOYED | BUSINESS | UNEMPLOYED

Loan Purpose (exact values):
  HOME | PERSONAL | BUSINESS | EDUCATION | VEHICLE | OTHER

Loan Amount:
  Minimum: 10,000
  Maximum: 5,00,00,000

Tenure:
  Minimum: 6 months
  Maximum: 360 months

Monthly Income:
  Minimum: 1000

Date of Birth Format:
  YYYY-MM-DD (example: 1998-05-15)

Decision Types (exact values):
  APPROVED | REJECTED

Document Status (exact values):
  VERIFIED | REJECTED

SECTION 8: COMMON ERRORS AND FIXES
================================================================================

ERROR: 401 Unauthorized
FIX  : You are not logged in or token expired
       -> Login again and re-authorize in Swagger

ERROR: 403 Forbidden
FIX  : You don't have the required role
       -> Make sure you are logged in as admin for admin endpoints
       -> Add X-User-Role: ROLE_ADMIN header

ERROR: 404 Not Found
FIX  : Wrong URL or service not running
       -> Check docker ps to see if service is running
       -> Check the endpoint URL is correct

ERROR: 400 Bad Request
FIX  : Validation failed
       -> Check the request body matches validation rules above
       -> Check password meets requirements

ERROR: 503 Service Unavailable
FIX  : Service not registered with Eureka yet
       -> Wait 30 seconds and try again
       -> Check http://localhost:8761 to see if service is registered

ERROR: 500 Internal Server Error
FIX  : Check service logs
       -> docker logs <service-name> -f

ERROR: Email already registered
FIX  : Use a different email or wipe data
       -> docker-compose down -v
       -> docker-compose up -d
