               ▼
┌─────────────────────────────┐
│     API Gateway :8080       │
│     JwtAuthFilter.java      │
│   reads Authorization header│
│   validates JWT signature   │  ← checks secret key
│   checks token not expired  │
│   extracts email + role     │
│   adds headers:             │
│   X-User-Email: user@...    │
│   X-User-Role: ROLE_USER    │
│   routes to correct service │
└──────────────┬──────────────┘
               │ request forwarded with X-User-Email, X-User-Role headers
               ▼
         Microservice receives request
         reads X-User-Email for identity
         reads X-User-Role for authorization

COMPLETE DATA FLOW SUMMARY
================================================================================

DATABASES (separate per service):
  finflow_auth        → users table
  finflow_application → loan_applications + outbox_events tables
  finflow_document    → documents table
  finflow_admin       → decisions table

SYNCHRONOUS CALLS (HTTP - caller waits):
  admin-service → application-service (get applications)
  admin-service → auth-service (get users)
  admin-service → application-service (update status after decision)

ASYNCHRONOUS CALLS (RabbitMQ - caller does NOT wait):
  application-service → RabbitMQ → admin-service
  (when application is submitted)

SECURITY FLOW:
  Angular → JWT in header → API Gateway validates → injects X-User-Email/Role
  → Microservice trusts headers from gateway

CONFIG FLOW:
  Service starts → fetches config from Config Server → connects to DB/RabbitMQ

DISCOVERY FLOW:
  Service starts → registers with Eureka → Gateway uses lb:// to find services
